CSS Display Attribute: Don't Leave Home Without It!

11/8/2015

Everything in CSS controls how things display, so why is there an attribute called "display", and what does it do?

When you load an webpage, all the elements on that page line up, starting at the top left corner of the page. It's sort of like lining up in the jetway to get on a plane. All elements have a default for how they behave in line. Things like bullet points don't want anything next to them, so their display behavior is "block". But normal text, like a link or a span tag, those expect to be used in the context of a paragraph where text will continue on the same line, so their behavior is "inline".

But you can change this behavior, and tell a span to display as a "block", which would be one way of adding a linebreak after it. Or telling a div to display as inline-block, so that it still takes up a block of space, but allows other elements next to it. There's also a "none" value, which makes it so that the element doesn't display or take up any space at all, something you might use to dynamically add and remove elements from view.

This is the tip of the iceberg though. There are display values for lists, tables, etc., which is how those HTML elements display by default, but you could go crazy and make your lists display like tables and vice versa, if you had some good reason for wanting to do that. For most purposes, you can probably get by with "block", "inline", "inline-block", and "none".
