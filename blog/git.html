The Magic of Version Control: Git, GitHub, and the Multiverse

10/21/2015

Why do you need version control?

You, yes you, need version control. And once you understand how useful version control is, you'll be wishing you could have it in aspects of your life beyond programming.

Version control is time travel.

Remember that one time in 8th grade when you played video games instead of studying for the history test the next day? Imagine if you could go back and correct that error. No one gets everything right on the first try. But with a version control system, you can figure out where a bug got introduced, and look at what your code was like before that point, and roll back to it if necessary.

Version control is access to parallel universes.

You aren't sure whether you want to continue your career in politics, or spend all your time carving scary faces into wood blocks, but you know you can't do both. Imagine if you could explore both options before you had to pick? Version control allows you to maintain distinct branches of your code, where you can safely go out on a limb without making a final decision.

Version control is also an all-seeing eye.

You failed chemistry lab because your lab partner added the wrong amount of hydrochloric acid. But once the ingredients were mixed, there was no way to prove that you weren't the one who did it. At least you had safety goggles on! With version control, you can collaborate with others, and track the changes each person is making and why they're making it, as well as having someone in control of what changes make it into the final product.

Convinced now? Then let's talk about the two parts of a very common version control system.

Git

Git is your local version tracker. It sits next to you and keeps track of all the changes you make. All the code for your project gets stored in a directory, and git keeps track of what changes you make to files in that directory. You can create code branches to try something out without messing up your master copy. Git lets you switch between them, without having to manage all those copies of the code yourself.

GitHub

As the name suggests, GitHub can function as a central online repository for your code and the history of your code changes, and those of other people contributing to the same code. It can also be how you release your code to the public, or as a backup in case you drop your laptop into lava.
