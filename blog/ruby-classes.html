Ruby Classes Are Worth Waking Up For

11/28/2015

In Ruby, you can create your own classes of objects, and give those objects their own methods (things they can do), and their own instance variables (attributes). Here's one example of why that's useful.

Let's say you want to create an object class of AlarmClock. For any alarm clock to work, it needs to be have a time that it goes off, and it needs to remember that time, even if you don't.

Here's how we could write our class in Ruby:

class AlarmClock
  
  def initialize (alarm_time)
    @alarm_time = alarm_time
  end

  def ring? (time_now)
    if time_now == @alarm_time
  end

end

This is great, because now we can set an alarm for when to get up in the morning:

morning_alarm = AlarmClock.new("7:45 AM")

Our morning_alarm will remember what time we want to get up, so if we're in bed and want to know whether it's time to get up yet, we can just roll over and type this into our Ruby console:

> p morning_alarm.ring?("7:30 AM")
false

Oh good, we can sleep a little more. But even better, we can create multiple alarm clocks, each set to different times. Each alarm clock will remember what time it's set to. In the meantime, we can work on other things, run other Ruby methods, and our alarm clocks will remain untouched.

If you had tried to keep track of your alarm time yourself, and made it a local variable instead of an instance variable, you'd have to define it at the console level and pass it into any method you want to have access to it. And then if any methods changed the value of that variable, you might wake up at the wrong time, or not at all.

This is just a tiny, not very functional example. There's a lot more you can do with classes and their instance variables in Ruby. You can create classes contained within other classes. You can have your object create another set of objects. You can also modify the standard Ruby classes, though it's not generally a good idea. The only limit is your imagination.
