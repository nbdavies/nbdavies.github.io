Ruby Enumberable Methods: #group_by

11/20/2015

Ruby has certain classes of objects called "enumerables" meaning essentially that these objects have multiple values, but the number of values they have is...enumerable. You could enumerate it or count it up, if you so desired. The most basic of these is a range of numbers like 1 to 6, which would have 6 values in it. But it also includes arrays and hashes.

One of the methods available to use on enumerables is #group_by. This method goes through each item in your object, evaluates it using a block of code, and creates a hash where the key is the result of the code block, and the hash value is an array of the items that got that result.

If you didn't follow that, don't worry. Let's look at an example. You have a list of songs, and you want to figure out what genre they are:

songs = ["Come Away With Me", 
         "Moon River",
         "Smells Like Teen Spirit", 
         "Sandstorm",
         "Song 2"]

Let's say you have a method to calculate the BPM of each song, and you figure out that songs with slow BPMs are jazz, songs with high BPMs are techno, and songs with middling BPMs are rock. You and I know that this is not universally true, but in this example, perhaps these are the only songs you've ever heard.

def genre (song)
  if song.bpm > 130
    return "Techno"
  elsif song.bpm < 90 
    return "Jazz"
  else return "Rock"
  end
end

Now you can create your separate lists like this:

songs_by_genre = songs.group_by {|song| genre(song)}

And you get this totally accurate and comprehensive categorization of songs by genre:

{"Jazz"=>["Come Away With Me", "Moon River"], 
"Rock"=>["Smells Like Teen Spirit", "Song 2"],
"Techno"=>["Sandstorm"]}

As you can see, you can use #group_by to sort a list of things into distinct groups. The method #sort_by is similar to #group_by, except after evaluating the code block for each item, instead of the hash I just showed you, it puts all the items in an array, ordered based on the output of the code block. But that output value (the genre in our example) isn't part of the array, so if you used #sort_by, you'd have a list of songs arranged by genre, but there wouldn't be any break between the jazz songs and the rock songs.
